# 政府需求診斷手冊

## 這本手冊要解決的問題
當客戶講一段需求，你要能做到：
- **快速判定這是哪一類系統（系統原形）**
- **立刻補齊常被漏掉、但會卡驗收/稽核/上線的需求**
- **主動提醒維運與總持有成本（TCO）的大頭與責任切分**
- **用專業話術帶領討論，而不是被動記錄需求**

## 最推薦的使用路徑（照這個順序走）
- **Step 1：先判定原形**：`02-系統原形庫-10種內/00-總覽-關鍵字到原形.md`
- **Step 2：打開該原形卡**：在 `02-系統原形庫-10種內/` 找對應檔案
- **Step 3：補齊隱性需求**：`03-隱性需求清單/00-隱性需求總表-可複製清單.md`
- **Step 4：把 TCO 與責任談清楚**：`04-維運與總持有成本-TCO/00-TCO總覽-構成.md`
- **Step 5：會議上遇到紅旗就用**：`05-風險與紅旗-RedFlags/00-紅旗總表-一句話提醒.md`
- **Step 6：會後直接交付文件**：`06-輸出物模板-可直接交付/00-需求診斷輸出範本.md`
