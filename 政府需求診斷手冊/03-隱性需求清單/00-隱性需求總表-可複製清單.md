# 隱性需求總表（可複製清單）

## 這份清單的定位
這章整理的是政府資訊專案最常「沒寫進需求、但最後一定會被追問」的項目。你可以把它當成：
- **售前提問清單**：把需求談完整、提早排雷
- **需求補齊清單**：把非功能需求與治理需求寫進規格
- **驗收與維運準備清單**：避免上線後才補洞

## 使用方式（建議流程）
- **第 1 步：判定系統原形**：先到 `02-系統原形庫-10種內/00-總覽-關鍵字到原形.md`
- **第 2 步：從下方清單勾選**：至少把「權限、稽核、例外處理、驗收」四類談清楚
- **第 3 步：落到可驗收**：每個勾選項目都要有「交付物」或「可測試條件」

## 哪些原形最常需要哪些隱性需求（快速對照）
- **進件審核／案件生命週期**：權限矩陣、稽核證據包、文件產製/簽章、例外/補償、驗收與SLA
- **帳籍／名冊／資產台帳**：權限矩陣、稽核（不可覆寫）、資料治理（主檔/口徑）、例外/補償
- **資料匯流／跨機關介接**：介接契約與版本、例外/補償（重試/補送/對帳）、稽核、驗收與SLA
- **公眾服務入口／CMS**：權限（審稿/發布）、稽核（發布留痕）、驗收（無障礙/資安/可用性）
- **身分與權限／SSO**：權限矩陣、稽核、介接契約（SSO協定/憑證）
- **文件／公文／電子簽章**：稽核證據包、文件產製/簽章、權限（調閱）、資料治理（保存年限）
- **資料平台／BI／AI**：資料治理（口徑/字典/品質）、權限（去識別/遮罩）、介接契約、例外/補償
- **雲端平台底座／代管**：驗收與SLA（監控/備援/演練）、例外/補償（事件處理）
- **資安／SOC／合規**：稽核（留存/證據）、驗收（弱掃滲測/修補）、權限（調閱）
- **GIS／IoT／指揮中心**：介接契約（設備/資料流）、資料治理（保存/品質）、例外/補償（斷線/失真）

## 可複製勾選清單（建議直接貼到需求文件）

### A. 權限與角色矩陣（必做）
- [ ] 角色清單（至少：承辦/覆核/主管/稽核調閱/系統管理者；視情境含民眾/承商）
- [ ] 權限維度：功能權限、資料列級權限（同案不同人看不同）、欄位級權限（敏感欄位）
- [ ] 權限申請/核可/到期收回流程（含臨時權限、代理）
- [ ] 特權帳號（最高權限）使用規則與稽核
- [ ] 驗收方式：用「角色測試清單」驗證每個角色的可見/可編輯/可決行

延伸閱讀：`01-權限與角色矩陣.md`

### B. 稽核軌跡與證據包（必做）
- [ ] 稽核事件清單（登入、權限變更、敏感操作、資料更正、核定/決行…）
- [ ] 不可否認與不可竄改（至少要能追溯；必要時需完整留痕/簽章）
- [ ] 查核輸出（案件/資料的「證據包」一鍵匯出：摘要、歷程、附件清單）
- [ ] 留存年限、調閱權限、調閱也要留痕
- [ ] 驗收方式：指定案例可匯出證據包；稽核事件可查、可篩選、可匯出

延伸閱讀：`02-稽核軌跡與證據包.md`

### C. 文件產製與簽章（常見卡關點）
- [ ] 文件類型清單（函稿/核定書/證書/報表/對外通知…）
- [ ] 模板與版面（固定欄位、動態欄位、條碼/水印、編號規則）
- [ ] 簽核/用印/電子簽章需求（是否需時間戳、驗章、撤銷）
- [ ] 文件版本控管與保存/封存
- [ ] 驗收方式：提供樣張；指定資料可產出一致版面；簽章可驗證

延伸閱讀：`03-文件產製與簽章.md`

### D. 資料治理與口徑（避免上線後一直對資料）
- [ ] 主檔/代碼表清單（組織、分類、狀態、地點…）與管理方式
- [ ] 指標/欄位口徑（資料字典、KPI字典、例外計算方式）
- [ ] 資料品質規則（缺值/重複/範圍/一致性）與品質報告
- [ ] 個資/敏感資料處理（遮罩/去識別、最小權限、留存/刪除）
- [ ] 驗收方式：資料字典交付；品質規則可執行；抽樣對帳一致

延伸閱讀：`04-資料治理與口徑.md`

### E. 介接契約與版本管理（跨機關/跨系統必做）
- [ ] 介接清單（API/檔案/DB/訊息）與責任窗口
- [ ] 資料契約（schema、代碼表、版本、相容策略、棄用/退場流程）
- [ ] 認證授權（憑證/VPN/白名單）、輪替週期與停機窗口
- [ ] 錯誤碼、重試策略、冪等性、速率限制
- [ ] 驗收方式：模擬對方斷線/改欄位；系統能告警、能補送/回復

延伸閱讀：`05-介接契約與版本管理.md`

### F. 例外處理與補償機制（不上這個，上線後就是救火）
- [ ] 失敗分類（可重試/不可重試/需要人工介入）
- [ ] 補送/回放/重跑（批次ID、死信匣、重處理介面）
- [ ] 對帳機制（來源 vs 落地 vs 報表一致性）
- [ ] 人工更正流程（更正單/調整單＋稽核）
- [ ] 驗收方式：指定失敗案例可復原；資料可對帳；更正可追溯

延伸閱讀：`06-例外處理與補償機制.md`

### G. 上線驗收與可用性（SLA）
- [ ] 非功能需求：效能/容量、可用性、資安、備份、RTO/RPO
- [ ] 監控告警（SLI/SLO）、值班與通報流程
- [ ] 備援/災難復原演練（演練頻率、演練證據）
- [ ] 驗收方式：壓測/弱掃滲測報告；DR演練報告；監控儀表板

延伸閱讀：`07-上線驗收與可用性SLA.md`
