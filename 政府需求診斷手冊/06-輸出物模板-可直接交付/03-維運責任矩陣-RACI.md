# 維運責任矩陣（RACI）

## 用法
- R（Responsible）：實作/執行者
- A（Accountable）：最終負責/拍板者（通常只有一個）
- C（Consulted）：被諮詢者
- I（Informed）：被告知者

## RACI 表（模板）
| 項目 | 內容 | R | A | C | I |
|---|---|---|---|---|---|
| 監控告警 | 儀表板/告警規則/告警渠道 |  |  |  |  |
| 事件處理 | P1~P4分級/回應時間/通報 |  |  |  |  |
| 部署上版 | 佈署/回滾/停機公告 |  |  |  |  |
| 備份與還原 | 備份策略/還原演練 |  |  |  |  |
| DR演練 | RTO/RPO演練與留證 |  |  |  |  |
| 弱點修補 | 弱點分級/修補SLA/複測 |  |  |  |  |
| 介接維運 | 憑證輪替/欄位改版/對帳 |  |  |  |  |
| 資料更正 | 更正單/調整單/稽核 |  |  |  |  |
| 客服回應 | 民眾留言/表單回應SLA |  |  |  |  |
